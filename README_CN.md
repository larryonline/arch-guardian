# Arch Guardian

架构守护工具 — 一个 Claude Code 技能，用于对项目架构进行结构化审查，防止架构劣化。

## 快速安装

```bash
claude install-skill https://github.com/larryonline/arch-guardian/raw/main/arch-guardian.skill
```

## 功能

**手动审查** — 在 Claude Code 中提及架构相关话题即可触发，执行三层结构化审查：

| 层级 | 审查项 |
|------|--------|
| 架构层 | 分层清晰度、目录结构、全局依赖、变更影响 |
| 模块层 | 模块边界、模块耦合、可替换性、数据流向 |
| 组件层 | 单一职责、命名可读性、依赖方向、可扩展性 |

**变更影响速检** — 在 `git commit` 前自动触发。对暂存区的变更进行轻量级扫描（层级归属、跨层依赖、模块边界、接口变更），有风险时输出一行提示，无风险则静默通过。

## 使用

**手动审查** — 提及架构相关话题即可触发：

- "帮我审查一下这次变更的架构影响"
- "项目结构感觉有点乱，帮我梳理一下"
- "我准备新增一个模块，先做个设计审查"

**速检** — 在每次 `git commit` 前自动运行。首次使用时，技能会询问是否自动配置 hook。

## 工作原理

1. **Hook 配置** — 首次使用时检测速检 hook 是否已配置，并提供自动配置
2. **知识发现** — 扫描项目中的架构文档、约定、技术栈信息和 ADR，持久化到 `.claude/arch-guardian/guardian-knowledge.md`
3. **范围适配** — 根据变更规模自动调整审查深度（聚焦/标准/全面）
4. **结构化报告** — 输出每个审查项的通过/警告/不通过判定，附具体发现和可行建议
5. **历史追踪** — 将报告保存到 `.claude/arch-guardian/`，追踪架构演进轨迹
