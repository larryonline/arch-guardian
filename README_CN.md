# Arch Guardian

架构守护工具 — 一个 Claude Code 技能，用于对项目架构进行结构化审查，防止架构劣化。

## 快速安装

```bash
claude install-skill https://github.com/larryonline/arch-guardian/raw/main/arch-guardian.skill
```

## 功能

- **设计审查（前置）** — 在新功能/模块实现前，评估设计方案对整体架构的影响
- **实现审查（后置）** — 代码完成后，审查实现是否符合架构规范
- **架构文档同步** — 检查架构文档与代码的一致性

## 审查维度

| 层级 | 审查项 |
|------|--------|
| 架构层 | 分层清晰度、目录结构、全局依赖、变更影响 |
| 模块层 | 模块边界、模块耦合、可替换性、数据流向 |
| 组件层 | 单一职责、命名可读性、依赖方向、可扩展性 |

## 工作原理

1. **知识发现** — 首次使用时，扫描项目中的架构文档、约定、技术栈信息和 ADR
2. **范围适配** — 根据变更规模自动调整审查深度（聚焦/标准/全面）
3. **结构化报告** — 输出每个审查项的通过/警告/不通过判定，附具体发现和可行建议
4. **历史追踪** — 将报告保存到 `.claude/arch-guardian/`，追踪架构演进轨迹

## 使用

安装后，在 Claude Code 中提及架构相关话题即可自动触发，例如：

- "帮我审查一下这次变更的架构影响"
- "项目结构感觉有点乱，帮我梳理一下"
- "我准备新增一个模块，先做个设计审查"
